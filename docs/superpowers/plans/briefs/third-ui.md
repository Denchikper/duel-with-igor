# Бриф для третьего агента — трек «Интерфейс»

> Скопировать всё, что ниже разделителя, целиком в чат с агентом.

---

Мы делаем Telegram Mini App для хакатона: «Дуэль с Игорем» — асинхронные дуэли по задачам ЕГЭ по информатике. Игрок проходит 7 вопросов на время, кидает другу ссылку-вызов, друг проходит те же вопросы, дальше экран сравнения. Персонаж-ведущий — Игорь Линьков, преподаватель информатики.

Стек: React 19 + TypeScript + Vite. Ты отвечаешь за **весь визуальный слой**. Твоя зона — только каталог `src/ui/`. Файлы за его пределами не создавай и не редактируй, над ними параллельно работают двое других. Работай в ветке `track/ui`.

## Железное правило

Ни один файл в `src/ui/` не импортирует `supabase`, не делает `fetch` и не содержит игровой логики. Данные приходят пропсами, события уходят колбэками. Благодаря этому ты пишешь весь интерфейс, не дожидаясь, пока будет готово ядро.

Пропсы всех компонентов **уже определены** в `src/contracts.ts`. Прочитай файл. Не меняй его и не отклоняйся от сигнатур — под них уже пишется ядро.

## Визуальное направление

Айдентика хакатона: тёмный фон, кислотно-зелёный акцент, терминальная эстетика — сетка, моноширинные подписи, ощущение консоли. Это должно чувствоваться, но не мешать читать задачу с телефона.

Тема (`src/ui/theme.css`, импортировать в `src/main.tsx`):

```css
:root {
  --bg: var(--tg-theme-bg-color, #0d0f0a);
  --text: var(--tg-theme-text-color, #eaffd0);
  --muted: #7c8a63;
  --accent: #b4ff39;
  --secondary: #8b5cf6;
  --danger: #ff5c5c;
  --card: rgba(180, 255, 57, 0.06);
  --border: rgba(180, 255, 57, 0.25);
}

body {
  margin: 0;
  background: var(--bg);
  color: var(--text);
  font-family: -apple-system, system-ui, sans-serif;
}
```

Фон и текст берутся из переменных темы Telegram с падением на тёмную — приложение должно оставаться читаемым и на светлой теме клиента.

Верстай под ширину 390px. Все интерактивные элементы — минимум 48px по высоте касания.

## Задача 2 — базовые компоненты

**`Button.tsx`** (`ButtonProps`) — `primary` заливается акцентным зелёным с тёмным текстом, `secondary` — прозрачный с рамкой. Во всю ширину, радиус 14, отступ сверху 12.

**`IgorBubble.tsx`** (`IgorBubbleProps`) — пузырь с репликой и аватаром Игоря. `mood` меняет цвет рамки: `happy` — акцентный, `disappointed` — красный, `smug` — фиолетовый, `neutral` — приглушённый. Реплика меняется часто, поэтому смену текста анимируй коротким fade, иначе текст дёргается.

**`Timer.tsx`** (`TimerProps`) — горизонтальная полоса, заполнение `remainingMs / totalMs`. Ниже 5 секунд — красная и пульсирует. Рядом число секунд моноширинным шрифтом.

**`ProgressBar.tsx`** (`ProgressBarProps`) — семь сегментов в ряд. Верные ответы зелёные, неверные красные, текущий с рамкой, будущие приглушённые. `answers` — массив длины 7, где `null` означает «ещё не дошли».

## Задача 3 — карточка вопроса

**`QuestionCard.tsx`** (`QuestionCardProps`) — самый важный компонент, он на экране бóльшую часть демо.

- Шапка: «Вопрос N из 7» и тема мелким моноширинным шрифтом.
- Текст вопроса крупно, от 18px, обязательно `white-space: pre-wrap` — во фрагментах кода на Python значимы отступы и переносы.
- Четыре варианта — крупные кнопки в столбик.
- `revealed === false`: нажатие вызывает `onSelect(index)`.
- `revealed === true`: верный вариант всегда подсвечен зелёным; если `selectedIndex` не совпал с верным — выбранный подсвечен красным; под вариантами появляется блок `explanation`.
- `disabled === true`: нажатия игнорируются.
- `selectedIndex === null` при `revealed === true` — это истёкшее время: подсветить верный вариант и показать разбор, ничего не помечая красным.

Проверь длинный вопрос с многострочным кодом — вёрстка не должна ломаться.

## Задача 4 — сравнение и таблица рекордов

**`ResultCompare.tsx`** (`ResultCompareProps`) — кульминация демо, его показывают на третьей минуте питча. Должен читаться с проектора за пять секунд.

- Шапка: две колонки с именами и итогом вида «5/7 · 38.4с» против «6/7 · 31.2с». Победитель подсвечен акцентом.
- Пузырь с `verdict` под шапкой.
- Таблица из 7 строк. В строке: номер, галочка или крестик у каждого игрока, время каждого в секундах. Тот, кто ответил быстрее **и верно**, помечен акцентом.
- Строка раскрывается по нажатию: текст вопроса, верный ответ, `explanation`. Это и есть польза приложения — разбор сразу после дуэли.
- `rival === null`: колонка соперника показывает «ещё не играл», таблица показывает только свои ответы, кнопка «Вызвать друга» выходит на первый план.
- Внизу две кнопки: `onShare` («Вызвать ещё») и `onRematch` («Новая дуэль»).

**`Leaderboard.tsx`** (`LeaderboardProps`) — нумерованный список. Аватар, при `photo_url === null` — кружок с первой буквой имени. Имя, счёт, время. Строка с `tg_user_id === meUserId` подсвечена и подписана «ты». Первые три места с медалями.

Проверь `ResultCompare` в обоих состояниях: с соперником и с `rival={null}`.

## Порядок и приоритет

Песочница → базовые компоненты → карточка вопроса → сравнение и рекорды.

Если времени в обрез, режь в таком порядке: медали в таблице рекордов, анимации, раскрытие строк в сравнении. Не режь: читаемость карточки вопроса и шапку экрана сравнения — это то, что видно в демо.

Коммить после каждого компонента, в ветку `track/ui`.
